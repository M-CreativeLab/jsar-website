# 打包（Package）

空间组件运行并调试通过后，可以将其打包成一个独立的文件，以便后续发布和真机调试。

> 当前打包的空间小程序需小于10MB，打包时若拓展发现空间小程序超过，会提示打包失败，开发者可以通过 [`gltf-transform`](https://gltf-transform.dev/) 工具进行模型压缩，或者通过其他方式优化你的包内容大小。

![howto-pack-project](./howto-pack-project.png)

如上图所示，在项目的空白处右键，选择 "JSAR: Package" 即可打包项目，然后就能在当前目录下看到一个 `{name}-{version}.idp` 的文件，其中，`name` 和 `version` 分别是配置在 `package.json` 中的名称和版本号。

> **什么是 `.idp`？**
>
> `.idp` 是 JSAR 的打包文件格式，它是一个压缩包，包含了空间组件的所有资源文件，包括模型、脚本、配置文件等。
>
> `.idp` 文件代表：Interactive Digital Product，即交互式数字产品，它表示一个在虚拟空间（元宇宙）中的一个跨场景的数字产品，用户可以与之交互。
