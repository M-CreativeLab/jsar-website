# 界面


拓展界面如下图所示，主要包含5个区域：

![](vscode-extension-gui.jpg)

1. 目录树，用于查看你项目的文件和代码
2. 编辑区，用于编辑单个文件的代码
3. 预览区，或场景预览，用于预览单个 XSML 文件
4. 文档树，用于查看当前运行的文档结构
5. 属性查看器，用于查看你选中的对象属性
6. 控制台，用于查看运行时日志以及 WebXR 调试器

## 目录树

如果你熟悉 VSCode 获取其他代码编辑器，一定不陌生，它是 VSCode 自带的功能，用于查看你的项目文件。

## 编辑区

同目录树一样，它也是 VSCode 自带的功能，用于编辑你的代码，但 JSAR 拓展会提供与 XSML/SCSS 语言相关的支持 —— 语法高亮与智能提示。

## 预览区

当你打开一个以 `*.xsml` 结尾的文件，会在右上角发现一个预览按钮，点击它就可以打开一个预览区，并读取你当前的 XSML 文件内容，用于渲染场景，开发者可通过它对场景进行预览。

## 场景树

当每次在预览区加载一个 XSML 时，会将中间的内容也同步给道 Scene Hierachy，用于查看当前场景的结构，开发者可通过它查看场景中节点的层级关系，以及节点的属性。

## 属性查看器

需要与场景树配合使用，通过场景树选择对象后，可在属性查看器中查看当前选中对象的属性，包含：

- 基础属性
- 位置、旋转、缩放属性
- 材质属性

> 未来会支持通过属性查看器修改对象属性，并同步到代码中。

## 控制台

控制台继承在 VSCode 自带的控制台中，作为新增的 Tab，目前支持以下功能：

- JSAR Console：用于查看 JSAR 应用的运行日志
- WebXR：集成并适配了 [meta-quest/immersive-web-emulator][] 用于在 VSCode 中调试 JSAR 应用的 WebXR 部分

[meta-quest/immersive-web-emulator]: https://github.com/meta-quest/immersive-web-emulator
